20260716 Cursor Rules、CamScanner OCR、Excel MCP 改表

延續近幾日 Cursor 工作流實作：把「要實際程式碼／不要空談」寫進規則、釐清 Pages 404、從 CamScanner 分享鏈拉 OCR，並裝上 Excel MCP 讓 Cursor AI 直接改表。

今日工作大要

  1. 精煉 Cursor AI 提示詞（繁中回覆、改碼只貼差分、禁止高層次空話），並寫入專案規則。
  2. 排查 Settings → User Rules「Failed to add rule」紅字：改走專案 .cursor/rules/*.mdc。
  3. 確認 GitHub Pages 對 directory/logs/*.html 的 404 多為佈署延遲，檔案在遠端且之後可開。
  4. 建立本機 MyKnowledgeBase；用 CamScanner 分享 API 拉清單與 OCR，彙整筆記。
  5. 安裝 excel-mcp-server，寫入 ~/.cursor/mcp.json，讓 Cursor AI 可讀寫 .xlsx。

一｜Cursor Rules（專案規則）

因 User Rules 雲端同步常失敗，改放專案檔（Always Apply）：

  • 路徑：.cursor/rules/no-high-level-fluff.mdc
  • 要點：一律繁體中文；改碼只給變更＋前後幾行上下文；要修正／解釋就給實際內容，不給「你可以這樣做……」空教學。

User Rules 若仍要加：登出再登入、Network Diagnostics、關 VPN；失敗就繼續用專案 .mdc。

二｜CamScanner → 知識庫

  • 分享頁是 SPA，HTML 幾乎無內容；改打 d82.intsig.net/sync/query_share_info_with_link 取文件清單。
  • OCR：POST /query_ocr_list?encrypt_id=…&doc_id=…（doc_id 為去掉 .jdoc 的檔名）。
  • 本機目錄：C:\Users\shaej\MyKnowledgeBase，目標是把多份掃描稿整理成單一 Markdown／筆記檔。

三｜Excel MCP（讓 AI 改表）

目標：在 Cursor Chat 直接讀寫 .xlsx（不需開 Excel 桌面版共編）。

  • 套件：excel-mcp-server（haris-musa/excel-mcp-server）
  • 啟動：python -m excel_mcp stdio
  • 設定檔：C:\Users\shaej\.cursor\mcp.json
{
  "mcpServers": {
    "excel": {
      "command": "C:\\Users\\shaej\\AppData\\Local\\Programs\\Python\\Python312\\python.exe",
      "args": ["-m", "excel_mcp", "stdio"]
    }
  }
}
  1. 完全重開 Cursor。
  2. Settings → MCP，確認 excel 綠燈。
  3. 用 Cursor 打開含 .xlsx 的資料夾。
  4. Chat 下指令，例如：「讀 Sheet1 前 20 列」「B 欄加總寫進總計列」。

注意：改表前先關閉 Excel 中同一個檔；重要檔先備份。若已有 chart／TalkToFigma 等 MCP，請合併進同一個 mcp.json，不要只留 excel 一顆。

成果對照

項目 狀態 說明
專案 Rules 完成 no-high-level-fluff.mdc alwaysApply
User Rules UI 可跳過 雲端 Failed to add rule → 用專案規則即可
CamScanner OCR 流程 打通 API 清單＋OCR；內容進 MyKnowledgeBase
Excel MCP 已安裝 重開 Cursor 後驗綠燈即可改表

建議下一步

  1. 把 chart／TalkToFigma／excel 三顆 MCP 合併進同一份 mcp.json 並 Refresh。
  2. 丟一份實際 .xlsx 給 Cursor，驗讀寫一輪。
  3. 把正式掃描 PDF／OCR Markdown 固定進知識庫＋掛 Rules。
