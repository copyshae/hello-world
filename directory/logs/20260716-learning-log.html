20260716 Cursor Rules、Excel MCP 投資分析與優先順序表

延續 Cursor 規則／CamScanner／Excel MCP；並用 Excel MCP 與 Python 解析 MY.xlsx，產出獲利前十檔、優先投資順序，寫回桌面「投資」資料夾。

今日工作大要

  1. 精煉 Cursor AI 提示詞，改走專案 .cursor/rules/*.mdc（User Rules 雲端同步常失敗）。
  2. CamScanner 分享 API → OCR → MyKnowledgeBase。
  3. 安裝 excel-mcp-server，讓 Cursor 可讀寫 .xlsx。
  4. 安裝 Cursor Excel 預覽擴充（XLSX Viewer／Office Viewer／Excel Viewer）。
  5. 解析 Desktop/投資/MY.xlsx：持倉、追蹤、交易損益、紅利；做圖表與集中度風險判讀。
  6. 依已實現獲利／勝率篩出優質基金十檔，排出優先投資順序。
  7. 產出交付檔 Desktop/投資/投資優先順序.xlsx（三個工作表）。

一｜Cursor Rules（專案規則）

  • 路徑：.cursor/rules/no-high-level-fluff.mdc（Always Apply）
  • 要點：繁體中文；改碼給差分；禁止高層次空話。

二｜CamScanner → 知識庫

  • 分享頁為 SPA；改打 query_share_info_with_link／query_ocr_list 取 OCR。
  • 本機：C:\Users\shaej\MyKnowledgeBase

三｜Excel MCP 與擴充

  • MCP：excel-mcp-server 寫入 ~/.cursor/mcp.json。
  • 擴充：muhammad-ahmad.xlsx-viewer、cweijan.vscode-office、grapecity.gc-excelviewer。
  • 注意：部分工作表 used range 異常大（如交易損益至 XEY），需限欄／限列或改用 Python 解析。

四｜MY.xlsx 投資分析（本段重點）

學到的要點

  • 歷史勝率高 ≠ 持倉結構健康；要同時看集中度與未實現損益來源。
  • 「優質基金」應以你帳戶真實已實現獲利排序，再疊配置上限，避免單一主題過重。
  • Excel 異常 used range 會拖垮 MCP；Python／限範圍讀取較穩。

建議下一步

  1. 依 投資優先順序.xlsx 第 1–5 順位設想買價，分批配置。
  2. 減碼黃金／原物料重複檔，釋出資金輪入收益成長與科技衛星。
  3. 把 chart／TalkToFigma／excel 合併進同一份 mcp.json 並 Refresh。
