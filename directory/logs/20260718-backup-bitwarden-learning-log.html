20260718 遠端保全與 Bitwarden 密碼管理

本日後段主線：讓 Cursor／知識庫努力在「電腦壞了」後仍可接續——Git 遠端備份、災難復原通則，以及用 Bitwarden＋Google Authenticator 保管私人機密。

一｜今日核心目標

  • 知識庫第一次完整推上私人 GitHub，避免本機單點故障。
  • 把「主動 commit／push、機密不進庫」寫成各專案通則。
  • 選定並完成 Bitwarden：三筆關鍵機密、手機驗證器 2FA、復原碼存檔。

二｜遠端保全（Git）

項目做法
遠端倉庫私人庫 copyshae/super-life-code（首次 commit＋push）
專案規則.cursor/rules/disaster-recovery.mdc；並設 Cursor User Rule
機密隔離今日行程/auth.json、config.json、ICS 快取等維持 gitignore
收錄索引日期｜索引頁已上線 hello-world Pages
通則重點：有實質產出後主動 commit＋push；結論寫進文件，不只留在聊天。

三｜Bitwarden 密碼管理器

選定免費雲端版 Bitwarden（跨 Windows／iPhone；適合存登入與安全筆記）。

已完成說明
帳號＋三筆今日行程解鎖密碼、私人日曆 ICS、GitHub
兩步驟登入驗證器已啟用；手機用 Google Authenticator 產生 6 碼
復原碼已下載存檔（弄丟手機時用）
之後怎麼存網站帳密用「登入」；備註型機密用「備註」；銀行帳密可存（建議搭配銀行本身驗證）

學到的操作點

  • Bitwarden「密碼庫」App 的「＋」是存帳密，不是掃帳號 2FA；帳號 2FA 要用 Authenticator／Google Authenticator。
  • 「驗證碼」欄位填 6 位數字，不要貼粉紅長串金鑰。
  • 系統相機掃 QR 通常不會自動加入驗證器；要在驗證器 App 內掃或「輸入設定密鑰」。
  • 金鑰／QR／6 碼勿貼到聊天；若曾外洩，可停用後重開 2FA 換新金鑰。

四｜換機復原口訣

  1. git clone 知識庫遠端倉庫
  2. 登入 Bitwarden（主密碼＋Authenticator 6 碼）還原行程密碼與 ICS
  3. 同一 Cursor 帳號登入帶回 User Rules
