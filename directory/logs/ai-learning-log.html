Cursor Vibe Coding 初學者課程大綱

用自然語言與 AI 對話，在 Cursor 裡做出第一個小作品

授課對象： 完全沒寫過程式的初學者

工具重點： Cursor（內建 AI 的程式編輯器 / IDE）

課程目標： 學會用清楚的描述引導 AI、讀懂 AI 產出的內容、實際測試與修正，完成可展示的小專案

建議時長： 8–10 堂課（每堂約 1.5–2 小時）

什麼是「Vibe Coding」？

Vibe Coding 是一種開發方式：你用日常語言描述想做的事，AI 幫你寫程式，你再一起調整到可用為止。

你不必先背語法。你需要學會這四件事：

  1. 把目標說清楚
  2. 用白話讀懂 AI 寫了什麼
  3. 實際打開／執行，看結果對不對
  4. 請 AI 修正或改進

本課程教的就是：在 Cursor 裡穩定地重複這個循環，讓開發體驗更流暢。

如何使用本大綱

  • 請依章節順序進行；每一章都建立在前一章之上。
  • 每一章包含：學習目標、教學重點、AI 練習、小專案、過關檢查。
  • 鼓勵學員請 AI「用白話解釋」，而不只是「直接給完整程式」。
  • 課堂原則：先理解，再接受。不清楚用途的程式，先問清楚再使用。
  • 每堂課盡量讓學員帶著「能打開、能示範」的成果離開。

課程核心原則

  1. 先描述，再動手 — 用日常句子寫下想做什麼。
  2. 每堂課都用 Cursor — 以 Chat、Inline Edit、Composer / Agent 為主要工具。
  3. 把 AI 當開發夥伴 — 多問「為什麼」「改了什麼」「請簡單說明」。
  4. 一次只做一小步 — 一次加一個功能，比一次做全部更穩。
  5. 早點測試 — 跑不起來是有用資訊，不是失敗。
  6. 你負責方向 — AI 幫忙打字與建議，最終決定權在你。

第 0 章 — 認識 Cursor 與 AI 寫程式

學習目標

  • 用簡單話說出「IDE 是什麼」
  • 打開 Cursor，認得畫面主要區域
  • 理解 vibe coding 與「先背語法」的差別

教學重點

  1. 什麼是寫程式？（給電腦清楚、可執行的指示）
  2. 什麼是 Cursor？（能寫檔案、改檔案，並用 AI 協助的編輯器）
  3. 畫面導覽：檔案列表、編輯區、終端機、AI 對話面板
  4. 建立練習資料夾，並用 Cursor 打開
  5. 安全習慣：只在練習資料夾操作；不要貼上密碼、金鑰等機密資訊

AI 練習

  • 問 AI：「用完全不懂程式的人聽得懂的方式，解釋 Cursor 是什麼。」
  • 問 AI：「用一小段話說明什麼是 vibe coding。」

小專案

  • 在 Cursor 打開一個空資料夾，命名為 my-first-project
  • 請 AI 建立一個簡單的 README.md，寫上自己的名字與想學什麼

過關檢查

學員能打開資料夾、找到 AI 對話，並用 AI 建立第一個檔案。

第 1 章 — 跟 AI 說話：寫出清楚的提示詞

學習目標

  • 寫出具體、適合初學者的提示詞（prompt）
  • 告訴 AI 自己的程度與目標
  • 分辨「模糊提示」與「清楚提示」

教學重點

  1. 好的提示詞通常包含：

- 你是誰（初學者）

- 你想做什麼

- 看起來／用起來要怎樣

- 限制條件（盡量簡單、先一個檔案等）

  1. 不好的提示：「做一個網站。」
  2. 更好的提示：「我是初學者。請做一個單一 HTML 頁面，包含我的名字、一段自我介紹，背景用藍色。並用白話解釋每一段在做什麼。」
  3. 追問技巧：「把字變大。」／「加一個按鈕，點了會顯示歡迎訊息。」
  4. AI 做錯時：不要慌，把錯誤訊息或現象描述貼給 AI，請它修正

AI 練習

  • 全班一起把 3 句模糊提示改成清楚提示。
  • 同一需求問兩次：一次模糊、一次清楚，比較結果差異。

小專案

  • 用 Chat 產生一頁自我介紹（index.html）
  • 請 AI 用白話解釋每一段
  • 再用追問改顏色與標題

過關檢查

學員能寫出清楚的第一句提示，以及至少一次有效追問。

第 2 章 — 第一個能打開的網頁（用 AI 學 HTML）

學習目標

  • 理解網頁就是瀏覽器能打開的檔案
  • 認得基本元素：標題、段落、圖片、連結、按鈕
  • 能在瀏覽器預覽頁面

教學重點

  1. HTML 是什麼？（頁面的骨架／結構）
  2. 如何在瀏覽器打開 HTML（或使用簡易即時預覽）
  3. 請 AI 解釋它剛產生的 HTML
  4. 小改動：改文字、顏色、排版 — 先請 AI 改，再在檔案裡找到對應位置
  5. 存檔與命名習慣（例如 index.html）

AI 練習

  • 「像我從未寫過程式一樣，解釋這段 HTML。」
  • 「指出哪一行控制網頁標題。」
  • 「只改標題文字為……」

小專案

做一頁「我喜歡的事物」：

  • 一個標題
  • 三段短文或一個清單
  • 一張圖（或佔位圖）
  • 一個連結

過關檢查

學員能在瀏覽器打開頁面，並指出檔案中哪一段控制標題。

第 3 章 — 讓頁面更好看（用 AI 學 CSS）

學習目標

  • 理解 CSS 是「外觀」層（顏色、字體、間距）
  • 分得清「內容」與「長相」
  • 只改外觀時，會寫對應的提示詞

教學重點

  1. CSS 是什麼？（控制看起來的樣子）
  2. 建議使用獨立的 styles.css（保持簡單）
  3. 常見概念：顏色、字級、margin、padding、背景
  4. 提示句型：「內容不要改，只把外觀改成……」
  5. 手機也能看：請 AI 讓頁面在手機上好讀

AI 練習

  • 請 AI 為同一頁做兩種風格（例如：安靜風／鮮明風）。
  • 「用一句話解釋這條 CSS 規則在做什麼。」

小專案

美化第 2 章的頁面：

  • 更好的字體與間距
  • 清楚的頁首
  • 一個醒目的按鈕
  • 可存前後截圖，下課短暫分享

過關檢查

學員能請 AI 改外觀，並找到被改到的 CSS。

第 4 章 — Cursor 超能力：Chat、Inline Edit、Composer / Agent

學習目標

  • 依任務選擇合適的 Cursor AI 工具
  • 會用 Inline Edit 改「選取的那一段」
  • 理解較大任務可用 Composer / Agent，但仍要目標小而清楚

教學重點

  1. Chat — 提問、規劃、解釋、除錯
  2. Inline Edit（Ctrl/Cmd + K） — 只改選取範圍
  3. Composer / Agent — 跨檔案的較大任務（仍要用小步驟描述）
  4. 怎麼選：

- 「這是什麼意思？」→ Chat

- 「重寫這段按鈕區」→ Inline Edit

- 「再加一頁並互相連結」→ Composer / Agent

  1. 接受／拒絕變更：先看差異（diff），不要盲目全部接受

AI 練習

  • 選取一段內容，用 Inline Edit 改樣式。
  • 在 Chat 先只要計畫：「只列步驟，先不要程式碼。」

小專案

把個人頁變成兩頁小網站：

  • Home
  • About
  • 用 AI 加上連結，再自己點開確認

過關檢查

學員能有意識地分別使用 Chat 與 Inline Edit。

第 5 章 — 加上互動（用 AI 學簡單 JavaScript）

學習目標

  • 理解 JavaScript 讓頁面能回應點擊與輸入
  • 用白話描述互動行為
  • 在瀏覽器測試按鈕／輸入是否有效

教學重點

  1. JavaScript 是什麼？（行為／互動）
  2. 生活化例子：點按鈕 → 顯示訊息；輸入名字 → 打招呼
  3. 提示句型：「當使用者點 X，就做 Y。請保持簡單。」
  4. 如何描述問題：「我點了按鈕沒反應」＋你原本期望什麼
  5. 用白話理解瀏覽器錯誤（紅字）：請 AI 解釋

AI 練習

  • 請 AI 加一個打招呼按鈕，再請它逐行解釋新程式。
  • 故意改錯一個名稱，再請 AI 幫你找問題。

小專案

做「心情檢測」或「名字打招呼」：

  • 輸入框
  • 按鈕
  • 依輸入改變顯示訊息

過關檢查

學員能描述一個互動、請 AI 做出來，並在瀏覽器成功示範。

第 6 章 — 冷靜修問題（用 AI 除錯）

學習目標

  • 東西壞掉時有固定檢查順序
  • 會把有用資訊提供給 AI
  • 學會「一次只修一個問題」

教學重點

  1. 初學者除錯清單：

- 有沒有存檔？

- 瀏覽器開的是不是正確檔案？

- 實際發生什麼？原本期望什麼？

- 有沒有紅色錯誤訊息？

  1. 向 AI 求助的格式：目標／你做了什麼／你看到什麼／錯誤文字
  2. 避免「全部重寫」— 請 AI 給最小修正
  3. 先請 AI 用白話解釋錯誤，再改程式

AI 練習

  • 針對 2 個壞掉的示範頁，練習寫「好的問題回報」。
  • 「不要重寫整個檔案。告訴我最小的修法。」

小專案

  • 拿到一個稍微壞掉的舊專案（或故意弄壞）
  • 用清單 + AI 修到能用
  • 寫 3 點：哪裡錯、怎麼發現、怎麼修好

過關檢查

學員能清楚回報問題，並套用 AI 建議的小修正。

第 7 章 — 從頭到尾做一個迷你 App

學習目標

  • 寫程式前先做小計畫
  • 把功能拆成步驟
  • 完成一個可展示的初學者作品

教學重點

  1. 專案規劃模板：

- 給誰用？

- 使用者能做什麼？（最多 3 個功能）

- 需要哪些頁面／檔案？

  1. 建議建造順序：結構 → 外觀 → 互動 → 潤飾
  2. 先請 AI 給步驟計畫，再一步一步實作
  3. 範圍要小：能用的小作品，勝過半成品的大想法

建議題目（擇一）

  • 個人作品集首頁
  • 簡單食譜／興趣介紹頁
  • 待辦清單（新增／刪除）
  • 小費計算機或單位換算
  • 班級示範：「我們的小組介紹」小網站

AI 練習

  • 「我是初學者。請把這個專案拆成 5 步計畫。先不要寫程式。」
  • 每一步後：「只實作第 N 步。並說明你改了什麼。」

小專案

  • 完成選定的迷你 App，讓同學不用你幫忙也能打開使用
  • 寫簡短 README.md：它做什麼、怎麼打開

過關檢查

學員有一份可現場示範的完整小專案。

第 8 章 — 更流暢的 Vibe Coding 工作習慣

學習目標

  • 用小步、安全的方式持續開發
  • 專案資料夾保持整齊
  • 建立長期可重複的 AI 學習節奏

教學重點

  1. 檔案整理：清楚命名、一個專案一個資料夾
  2. （選修）版本概念：另存副本／簡單理解「快照」
  3. 建立個人「好用提示詞清單」
  4. 學習循環：請 AI 解釋 → 自己改一小處 → 請 AI 檢查
  5. 學習誠信：AI 能協助，但展示與評量時你仍要理解自己的作品
  6. AI 有極限：可能出錯，一定要自己測試

AI 練習

  • 建立個人 prompt-cheat-sheet.md，收錄 8–10 句好用提示。
  • 「用簡單問題考我：關於我的專案問 5 題。」

小專案

潤飾第 7 章作品：

  • 文字更清楚
  • 間距更好
  • 一項易用改善（例如更大按鈕、更清楚標籤）
  • 準備 2 分鐘示範稿

過關檢查

學員能示範作品，並回答「這段在做什麼？」至少兩處。

選修成果發表 — 秀出你做出來的東西

形式

  • 每人（或兩人一組）示範 3–5 分鐘
  • 在瀏覽器展示可運作的作品
  • 分享一句最有幫助的提示詞
  • 分享一個用 AI 修好的問題

可選加分方向

  1. 幫迷你 App 再加一個小功能
  2. 同一想法，重做更乾淨的版面
  3. 兩人一組互相協助改進（配對 vibe coding）

成功標準

  • 專案打得開、用得了
  • 能用白話說明目標
  • 能指出一處自己理解的 AI 協助修改

建議每堂課流程

  1. 暖身（10 分） — 打開上次作品，先做一個小成功
  2. 精講（20–25 分） — 一個新觀念 + 講師在 Cursor 現場示範
  3. 引導練習（25–30 分） — 跟著寫提示詞、一起操作
  4. 小專案（30–40 分） — 學員獨立做／改進
  5. 分享與反思（10 分） — 2–3 位學員展示；收集卡住的問題

講師提示

  • 投影 Cursor 畫面要夠大；一開始每個點擊都口頭說明。
  • 把錯誤平常化：「紅字是線索，不是災難。」
  • 鼓勵「我不會」— 那正是好的 AI 提問起點。
  • 優先流程：計畫 → 寫碼 → 解釋 → 測試，避免一次貼超大段程式。
  • 對零基礎學員，先停留在 HTML / CSS / 簡單 JavaScript。
  • 若學員陷入「一直重問 AI 仍無效」，停下來一起改提示詞。
  • 清楚的提示詞，與漂亮畫面一樣值得稱讚。

學員入門提示詞包（可直接發放）

可複製後改成自己的需求：

  1. 「我是完全初學者。請用白話解釋這個檔案在做什麼。」
  2. 「我想做 ___。請保持簡單。先用一個 HTML 檔。並解釋每一部分。」
  3. 「不要全部重寫。只改 ___。」
  4. 「先給我一步一步計畫，暫時不要程式碼。」
  5. 「我期望 ___，但實際看到 ___。錯誤訊息是：___。」
  6. 「像我剛學寫程式一樣解釋這個錯誤。然後給我最小的修法。」
  7. 「改善設計：間距清楚、字好讀、對比足夠。」
  8. 「只加一個功能：當使用者點 ___，就應該 ___。」
  9. 「用 5 個簡單問題考我對這個專案的理解。」
  10. 「在我上台示範前，列出我應該測試的項目。」

輕量評量建議

類型觀察重點
提示詞能力能把模糊需求改成清楚提示
工具能力會依任務使用 Chat 與 Inline Edit
理解力能用白話說明頁面中的 3 個部分
除錯能力能在 AI 協助下修好小問題
專案成果迷你 App 能運作，且符合自己說的目標

通過標準：學員能用 Cursor 做出、說明、改進 一個小專案 — 不是背誦語法。

課前準備檢查清單

  • [ ] 已安裝 Cursor 並完成登入
  • [ ] 有瀏覽器可預覽（Chrome / Edge / Firefox）
  • [ ] 有專用練習資料夾（例如 Documents/cursor-class）
  • [ ] （選用）截圖工具，方便前後對照分享
  • [ ] 已把課程檔與提示詞包发给學員

學習路徑總覽

章節主題主要成果
0環境與心態打開 Cursor；用 AI 建第一個檔案
1提示詞寫清楚提示與有效追問
2HTML第一個能在瀏覽器打開的網頁
3CSS用 AI 美化頁面
4Cursor 工具Chat、Inline Edit、Composer / Agent
5JavaScript做出簡單互動

7迷你 App完成端到端小專案

成果發表示範展示並反思

課程結束後可繼續的方向

  1. 同一專案重做一次：不急著接受第一版答案，先練更好的提示詞
  2. 在 Cursor 學簡單 Git（「存檢查點」的概念）
  3. 等 HTML/CSS/JS 熟悉後，再嘗試更進階工具
  4. 組成每週小型「一起做」練習團，持續練習同一個開發循環

大綱結束 — 請依班級節奏調整。對零基礎學員來說，理解深度比趕進度更重要。
